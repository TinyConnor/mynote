# git仓库链接多个远程仓库的方法
## 方法一：分别推送
1. `git remote -v`:查看当前本地仓库链接连接的远程仓库
2. `git remote add <仓库别名> <远程仓库地址>`:链接远程仓库，仓库别名要与上面查看到的不一样
3. `git remote -v`:检查是否多了新的链接
4. `git push <仓库别名> <分支名>`:分别推送不同的远程仓库
5. `git remote remove(或rm) <仓库别名>`:删除添加到的

## 方法二：一起推送
1. `git remote -v`:查看当前本地仓库链接连接的远程仓库
2. `git remote set-url --add <当前仓库别名> <远程仓库地址> `:链接远程仓库到当前仓库,当前仓库名为上边看到的如*origin*
3. `git remote -v`:检查当前仓库名是否多了一个新的push地址仓库
4. `git push`:一起推送即可
5. `git remote set-url --delete <当前仓库别名> <远程仓库地址> `:删除上面方式链接的仓库,当前仓库名为上边看到的如*origin*

## .gitignore
* 目录后加`/`
* `generated/`表示忽略所有这个目录，而`/generated/`表示仅忽略根结点
* 可以使用`& * !`等通配符
* .gitignore只能影响还未交由git管理的文件，如果想忽略已经版本化的文件通过`update-index --assume-unchanged`

## git add——注册修改
> 注册修改，将所做的修改纳入下次提交。

* `--all`:表示所有修改都纳入在内。

## git am——应用补丁文件
>与format-patch配合使用，将.patch文件应用到当前分支

* `<补丁文件.patch>`：应用单个补丁
* `</path/to/patches/*.patch>`:应用目录下的所有补丁（按字母顺序）
* `--3way --keep-cr --no-commit < patch-file.patch`:应用补丁但不提交
* `--3way`:使用三方合并，减少冲突概率
* `--abort`:取消当前打补丁操作
* `--skip`:跳过当前补丁

## git branch——分支
> 查看当前活跃分支

* `<新分支名>`:为当前提交创建分支
* `<新分支名> <散列值>`:指定某次提交为新分支的最新提交
* `<分支名> <新分支名>`:从现有分支中创建分支
* `-d <分支名>`:删除被终止的分支
* `-D <分支名>`:删除打开的分支
* `<分支名> <散列值>或<HEAD@{n}>`:恢复某个分支使用`git reflog`查看散列值
* `-r`:显示远程跟踪的分支
* `-v`:显示本地分支列表，并附带最后一次提交的哈希值和提交信息
* `-vv`:在 `-v` 的基础上，额外显示本地分支与远程分支的追踪关系，以及两者的提交差异（若存在）
* `-m <old-branch> <new-branch>`:更改分支名
*  `--set-upstream-to=<远程仓库名>/<远程分支名> <本地分支名>`:修改本地分支对应的远程分支

## git checkout——检出
> 后跟随散列值，恢复某一版本。

* `<散列值>：<文件名>`:恢复指定文件版本

> 后跟分支名切换某一分支，必须把工作区和暂存区清理干净

* `-b <分支名>`：创建并切换到新分支
* `-b <分支名> <散列值>`：恢复到⼀分⽀某⼀时间点的状态可以是删除的分支
* `--force <分支名>`:强制切换分支，原来工作区暂存区内容消失
---
## git cherry-pick——应用特定提交
>将一个或多个提交从其他分支复制到当前分支，生成一个新的提交(散列值不同，但内容相同)

* `<散列值>`:应用单个提交
* `<散列值1> <散列值2> <散列值3>`:应用多次提交
* `<start-conmmit>^..<end-conmmit>`:应用两次提交之间的所有提交(包含start，但不包含end)
* `-n(--no-commit)`:只更新工作区，不自动创建提交
* `-e(--edit)`:编辑提交信息
* `-x`:在提交中天健原提交的散列值，便于追踪
* `--abort`:取消当前cherry-pick操作

## git clean——删除未跟踪的文件
- `-d`:如果删除文件没有指定路径，这个指令会递归查找该文件
- `-f`:强制删除，必须加此命令才能真的删除
- `-n`:模拟删除，展示将要删除的文件但并不会真的删除
- `-x`:删除未跟踪的文件和gitignore忽略的文件
- `-X`:只删掉gitignore忽略的文件
- `-i`:交互式删除，显示候选文件列表并提供选择菜单

## git clone ——克隆仓库
- `<仓库位置>`:克隆本地仓库
- `<URL传输协议>:<仓库位置>`:克隆不同来源版本库，URL传输协议有：file、ssh、http、https、ftp、ftps、rsync
- `-b <指定分支> <仓库位置>`:克隆远程仓库指定分支，体积小，但保留其他远程分支的引用 *使用git branch -r可以查看*
- `--single-branch -b <指定分支> <仓库位置>`：克隆指定分支，体积更小，不存在对其他远程分支的引用，不能切换远程分支

## git config——配置git用户信息
`--<范围>`：可选全局、系统和局部(本地仓库) *优先级：局部>全局>系统*
	--system:系统，对所有用户的所有仓库生效 */etc/gitconfig或C:/ProgramData/Git/config*
	--global:全局，对当前用户的所有仓库生效 *~/.gitconfig或~/.config/git/config*
	--local(默认):仅当前仓库生效 *.git/config*
`--<范围> user.name "name"`:配置⽤户名
`--<范围> user.email "email"`:配置邮箱
`--<范围> credential.helper store`:保存⽤户名和密码
`--<范围> --list`:查看⽤户名和邮箱
`--<范围> --unset <属性>`:删除config指定键值对
`--<范围> --list --show-origin`:显示所有用户信息并附带来源

## git diff——比较两次提交差异
> 默认显示暂存区和工作区之间的区别

* `--staged或cached`:比较暂存区和版本库之间的区别
* `<散列值或分支、标签、HEAD等符号> <散列值或分支、标签、HEAD等符号>`:后跟两次版本的commit hash或分支、标签、HEAD等符号获取两次提交差别
* `<散列值>^!`:与上次提交进行比较
* `<散列值1> <散列值2> -<目录名或文件名>`:限制显示那些文件或目录差异
* `--stat <散列值1> <散列值2>`:显示每个文件修改数量

- `<版本库> <远程分支名>:<本地分支名>`:从远程分支拉取到本地分支,本地分支不存在就创建

## git format-patch——生成补丁文件
>将提交转换为补丁文件（`.patch` 或 `.diff`），便于在没有 Git 仓库的环境中应用，或通过邮件发送代码修改。

* `-n`:生成最近n个提交的补丁
* `<start-commit>..<end-commit>`:生成两个提交之间的补丁，包含start不包含end
* `<分支名>(origin/main)`:生成所有未推送到main的提交补丁
* `-o <dir>`:指定补丁文件的输出目录
* `--stdout`:将补丁输出到标准输出(可重定向到文件)
* `--subject-prefix="PATCH"`:自定义补丁右键的主题前缀
* `-s(--signoff)`:在补丁中添加你的签名

## git log——项目提交历史
* `-n 3`:限制输出，仅显示最近3次提交
* `--format`:控制输出，`--format=fuller`显示更多细节信息
* `--oneline`:显示概述信息
* `--stat`:显示被修改的文件
* `--dirstat`:显示被修改文件的目录
* `--shortstat`:显示项目中多少文件被修改，以及新增或删除多少文件
* `--graph`:显示个提交之间的关系
* `--merges`:显示合并历史
* `--first-parent`:显示第一父级提交历史
* `--decorate`：显示每个提交的引用，包括分支、标签等。
* `--all`：显示所有分支的提交历史，包括远程分支和分离头指针。
* `--decorate`:将标签页添加进日志输出中

- `<远程仓库名> --delete <远程分支名>`:删除远程分支

* `--comtinue`:解决冲突后继续
* `--skip`:跳过冲突提交

## git reset ——回收修改
* `--mixed <散列值>`:默认参数，重置分支指针(工作区不影响、暂存区会被覆盖)
* `HEAD <文件名>`:将文件在版本库和暂存区所做的修改回收到工作区
* `--hard <散列值>`:重置分支指针(工作区和暂存区会被覆盖)
* `--soft <散列值>`:重置分支指针(不影响工作区和暂存区)

## git restore ——恢复文件
- `-s(--source) <散列值> <文件>`:从指定提交中恢复文件
- `--worktree(默认)`:只恢复工作区
- `--staged`:只恢复暂存区
- `--worktree --staged`:同时恢复工作区和暂存区
- `--source <分支名> --worktree --staged <文件路径>`:从指定分支恢复指定文件

### dd--数据复制和转换
>低级数据复制：按块复制文件或设备（如硬盘、U 盘），适合创建磁盘镜像、写入系统安装介质。
数据转换：支持位翻转、大小写转换、去除回车等操作。

`if=<输入文件> of=<输出文件> [参数]`

`if=<文件>`:输入文件（如 `/dev/sda` 表示硬盘，`image.iso` 表示 ISO 文件）。
`of=<文件>`：输出文件（如 `/dev/sdb` 表示 U 盘，`backup.img` 表示镜像文件）。
`bs=<大小>`:块大小（如 `bs=4M` 表示每块 4MB，提高复制速度）。
`count=<数量>`:复制的块数（如 `count=100` 表示只复制 100 个块）。
`conv=<选项>`:转换模式（如 `conv=sync` 填充数据到块大小，`conv=notrunc` 不截断输出文件）。
`status=<进度>`：显示复制进度（如 `status=progress` 实时显示已复制字节数）。

### mount/unmount--挂载/卸载文件系统

>用于将文件系统挂载到指定的挂载点（目录），从而使存储设备（如硬盘、U 盘、CD-ROM）或网络共享资源可被访问

`-t <文件类型> -o <选项> <设备名> <挂载点>`
`-t`:文件系统类型
*  `ext4`：Linux 主流文件系统
- `ntfs`：Windows NTFS 文件系统
- `vfat`：FAT32 文件系统（U 盘、SD 卡常用）
- `iso9660`：CD/DVD 光盘文件系统
- `nfs`：网络文件系统（Network File System）
- `cifs`：Windows 共享文件夹（Common Internet File System）
- `tmpfs`：临时内存文件系统
`-o`:挂载选项
* `ro`:只读挂载（Read-Only）
* `rw`:读写挂载（默认）
* `remount`:重新挂载已挂载的文件系统（用于修改挂载选项）
* `noexec`:禁止执行文件系统中的可执行文件（增强安全性）
* `nosuid`:禁用 SetUID/SetGID 权限（防止提权攻击）
* `nodev`:不解释文件系统中的设备文件（如 `/dev/sda`）
* `sync`:同步写入（数据直接写入磁盘，而非缓存，安全性高但性能低）
* `async`:异步写入（默认，性能高但可能丢失未刷新的数据）
* `user`:允许普通用户挂载（需文件系统支持）
* `auto`:在系统启动时自动挂载（配合 `/etc/fstab` 使用）
* `uid=<用户ID>`:指定文件系统的所有者用户 ID
* `gid=<组ID>`:指定文件系统的所有者组 ID
* `loop`:将文件十位块设备(循环挂载)
`-a`:挂载 `/etc/fstab` 中列出的所有文件系统（常用于系统启动）
`-f`:模拟挂载（不实际执行，用于测试配置）
`-v`:详细模式（显示挂载过程的详细信息）
`-n`:不更新 `/etc/mtab` 文件（临时挂载，系统重启后失效）

`umount <挂载点>`:通过挂载点卸载
`umount <设备名>`:通过设备名卸载

### sed--流编辑工具
`-e '1i\a new line' <文件名>`:-e可省略，在第一行插入(insert)内容为"a new line"(只会在终端打印,不会真的修改文件内容)
`-i`：对文件生效,不在终端打印
`-e`:输出到终端 ，多个命令用多个-e
`-f <脚本文件> <文件名>`:多个命令写到脚本文件中，执行
`'<行号>a<\或空格><文本内容>'`：在某一行    新增     "内容"
`'<行号>c<\或空格><文本内容>'`:将文本内容整行覆盖到指定行
`'<行号>s/<被替换内容>/<替换内容>/'`:将指定行的**第一个**指定内容替换为新内容,没有指定行号就全局替换
`'<行号>s/<被替换内容>/<替换内容>/g'`:将指定行的**所有**指定内容替换为新内容,没有指定行号就全局替换
`-n '<行号>p'`:打印指定行内容
`'<行号>d'`：删除某行
